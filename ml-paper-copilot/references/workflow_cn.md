# ML Paper Copilot - 两阶段工作流

## 0. 总体规则

这是两阶段(2-Phase)全自动流程。
- Phase 0 仅此一次停顿
- Phase 1 一口气跑到底

## Phase 0: 初始化与拦截

必须收集并确认：
1. 目标期刊/会议或模板
2. 图表/结果材料(可声明暂无)
3. 结果摘要或 Markdown 碎片
4. .bib 文件
5. 文风偏好(默认客观、发现驱动、无元叙述)
确认后进入 Phase 1，不再停。

## Phase 1: 端到端暗箱执行

### Step A: 故事线设计
- 仅基于输入提取核心主张
- 生成 venue 对齐大纲和图表映射

### Step B: 状态机接力起草
- 建立内部 state(大纲、章节摘要、TODO)
- 分段起草，每段压缩成 5-10 行摘要写入 state
- 后续章节只依赖 state + 原始输入

### Step C: 内部魔鬼自审(隐式)
- 使用 Devil's Advocate + 方法学审查
- 发现 CRITICAL/MAJOR 自动重写并复查
- 不输出自审过程

### Step D: 模板注入 + 自动文献织入
- 选择 ml-paper-writing/templates 中的官方模板
- 直接注入模板，不手写 LaTeX 结构
- 引用仅来自 .bib
- 仅在条目存在时插入 \cite{}
- 缺失引用在 QA 中标注

## 最终输出

一次性输出：
1. 可编译 LaTeX 源码
2. 简短 QA 报告(风险点/缺失材料/未解决问题)
